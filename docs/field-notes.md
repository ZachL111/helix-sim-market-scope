# Field Notes

`helix-sim-market-scope` is easiest to review by starting with the fixture, not the prose.

The domain cases cover `input pressure`, `state drift`, `review cost`, and `decision risk`. They sit beside the smaller starter fixture so the project has both a compact scoring check and a domain-flavored review check.

The model rewards signal and confidence, then pulls back for drag. In this fixture, `edge` shows where that pullback matters most.

The extra check gives the repository a behavior path that can fail for a domain reason, not only a syntax reason.
